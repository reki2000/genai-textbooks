# やる夫教材カウント

集計規則の正本をこのスキルに置き、手計算や別実装を避ける。

## 集計する

リポジトリルートで次を実行する。

```bash
python3 .claude/skills/yaruo-count/scripts/count_textbooks.py
```

特定ファイルだけを集計する場合はパスを渡す。

```bash
python3 .claude/skills/yaruo-count/scripts/count_textbooks.py docs/books/example/README.md
```

機械処理には `--format json` を使う。別のPythonスクリプトから利用するときは、同スクリプトの `count_document(path)` を呼び出す。

## 文字数・行数の規則

- バイト数は元ファイル全体のUTF-8バイト数とする。
- 文字数はUnicodeコードポイント数とし、改行コードを含めない。
- `yaruo-format` で正規化済みの会話表記を入力とする。発言外枠のカギカッコと行頭の全角空白は、集計時に推測して除去しない。
- `**やる夫**：`、`**やらない夫**：` および `**やる夫**（老いて）：` のような注記付きの話者表示行を、文字数・行数の両方から除外する。登場人物欄と反復出現する話者ラベルの判定は `yaruo-format` と同じにする。
- セリフ本文はすべて集計対象に含める。発言の先頭に残された引用語など、意味を持つ `「`・`」` も文字数に含める。
- 行数は話者表示行を除外した後の物理行数とし、空行を含める。

## 読了時間の規則

- Markdownの `$...$` と `$$...$$` を、区切り記号を含めてTeX数式部分とする。
- コードフェンスとインラインコード内の `$` は数式として扱わない。
- TeX数式部分は10 Unicodeコードポイント/秒、それ以外の会話・本文部分は40 Unicodeコードポイント/秒として計算する。
- 両者の秒数を合計した後、分単位で四捨五入する。表示は `(0分)` 形式とする。

## 検証する

変更後は代表的な教材と全文書の両方でスクリプトを実行し、次を確認する。

- `math_characters + other_characters == characters`
- 秒数・分数が負でない
- 同じ入力から同じ結果が得られる
