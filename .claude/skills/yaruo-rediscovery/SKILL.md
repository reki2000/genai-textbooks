# やる夫式・再発見型教材ジェネレータ

生徒役が教師役との対話を通じて理論を「自力で再発明する」教材を作る。単なる解説の対話化ではない。核心は **「素朴な案を試す → 具体的反例で失敗する → 失敗の原因を言語化する → 正しい概念に到達する → 検算する」** のサイクルを、導入する概念ごとに回すことにある。

執筆前に必ず `docs/books/general-relativity.md`（一般相対性理論の完成例、約6万字）を読み、文体・密度・失敗と回収のリズムを体感すること。加えて、後述の「参照コーパス」からテーマ系統の近い完成例を1冊選んで読むこと。内容を流用するのではなく、呼吸と分量感を掴むために読む。

## 制作フロー

3段階で作る。いきなり本文を書かない。

**Step 1: プロット提示。** 幕構成レベルの粗いプロットを作り、ユーザーに提示して合意を取る。各幕の到達点（生徒が何に気づくか）を1〜2文で書く。総分量の目標（後述の基準内でどこを狙うか）と幕ごとの配分目安も添える（道具構築の幕が最厚になる旨）。

**Step 2: 躓きポイント表の設計（内部作業）。** 本文を書く前に、導入する概念ごとに次の3点を用意する。これが教材の品質を決める。

| 概念 | 生徒が試す素朴な案 | それを殺す具体的反例（トイ例） | 検算用の具体例 |

素朴な案は「初学者が本当に思いつきそうな、一見もっともらしい案」でなければならない。藁人形は禁止。反例は数行で計算でき、可能なら実際の数値が出るものにする。

**Step 3: 本文執筆。** Markdownファイルとして作成する。長い（数万字になる）ので分割して書いてよいが、一貫性を保つ。docs/book 配下に保存する。ファイル名は簡潔な英字ハイフン区切り。

## 分量と構成の基準

既刊22冊の実地レビューから得た数字。**目標総分量は5〜10万字**（完成例 general-relativity は6.0万字）。良書はほぼ全てこのレンジに収まっている。10万字を超えた本は、終盤の駆け足・カタログ化・同型ビートの摩耗が高確率で起きた（30万字級・42万字級の既刊で確認済み）。10万字を大きく超えそうな企画は、プロット段階で分冊か圧縮をユーザーに提案する。

幕ごとの配分目安：

| ブロック | 割合 | 備考 |
|---|---|---|
| 破綻・直観・跳躍（第1〜3幕相当） | 20〜30% | 素朴案の潰しはここが主戦場 |
| 道具の構築 | 40〜50% | 最厚。1節=1概念=1再発見サイクル |
| 要請からの導出（クライマックス） | 10〜15% | 条件列挙→棄却→一意化 |
| 検証・答え合わせ | 約10% | 数値照合と全行程の振り返り |
| 歴史・分派・応用の「ツアー幕」（置く場合） | **最大20%** | 超えると講義に退行する。作法はアンチパターン集を参照 |

節の粒度は「1節=1概念=1再発見サイクル」。1つの節で概念を2つ以上まとめて開示したくなったら、それは羅列化の前兆なので、節を割るか片方を捨てる。

## 全体構造テンプレート

6幕構成を基本形とし、テーマに応じて増減する。

1. **既存理論の破綻（動機）** ── 生徒が既知の理論同士の矛盾や説明不能な現象に気づく。ここで生徒が「安直な修正案」を出し、教師が反例で潰す。「小手先の修正では済まない、根本からの再構築が要る」と生徒自身に言わせる。
2. **物理的・概念的直観** ── 理論の核となる直観（思考実験など）。生徒は「それは自明では？」「似た状況でも同じでは？」と疑い、教師が対照実験・反例で「なぜこの性質だけが特別か」を掴ませる。
3. **概念的跳躍** ── 直観から核心アイデアへの飛躍。生徒は跳躍した直後に自分の言葉に自信を失い、素朴な疑問（「それは一体どういう意味だ？」）を出す。教師は簡単な具体例（歴史上の先人の道具）で足場を与える。答えを保留する伏線もここで植える。
4. **道具の構築（最厚、全体の約半分）** ── 理論を書くための数学・形式的道具を1つずつ組み立てる。各道具の導入で再発見サイクル（下記）を必ず回す。
5. **要請からの導出（クライマックス）** ── 生徒が「満たすべき条件」を番号付きで列挙し、単純な候補から順に試して条件違反で棄却し、最終的にほぼ一意に答えへ到達する。
6. **検証・答え合わせ** ── 既知の極限・実験事実・具体的な数値との照合。最後に全行程を振り返り、「引っかかった場所が全部要所だった」ことを生徒に言わせる。未解決問題への一言で締めると余韻が出る。

## 再発見サイクル（最重要）

概念を1つ導入するたびに、次の型を回す。省略した概念が1つでもあると「生徒が物分かり良すぎる」教材になる。

1. **素朴な案**：生徒がもっともらしい単純案を出す（「〜でいいお」「一番簡単なのを試すお」）。
2. **具体的反例**：教師が「では判定してもらおう」「反例をやる」と、数行で計算できるトイ例を出す。生徒が自分で計算して失敗を確認する。
3. **原因の言語化**：生徒が失敗の構造を自分の言葉で言う（「座標の取り方で結果が変わるのは物理としてありえないお」）。ここを教師に言わせない。
4. **修正・正しい概念**：教師は方向だけ示し、生徒が式や定義を組み立てる。概念の**命名は生徒が到達した後**に教師が行う。
5. **検算**：新しい道具を具体例に適用し、期待通り動くことを確認する。可能なら**合格例と不合格例の両方**で判定させる（例：測地線方程式を「放射直線→合格」「円→不合格」の両方でテスト）。

## 具体例（トイ例）の設計ルール

- **数行で計算が完結する**こと。長い計算は教師が結果だけ与え、生徒は代入・検算を担当する。
- **実際の数値**を出せるならそうする（「62,800 km のはずが 40,000 km しかない」のような、桁で驚ける数字が強い）。
- **反復利用できるトイ例を1つ育てる**。例示用の題材を幕をまたいで使い回すと、生徒の習熟が読者の習熟と同期する（完成例では「平面の極座標」が、計量→接続の計算→共変微分の検算→測地線の判定→曲率ゼロの確認、と5回再登場する）。新しい教材でも「単純だが非自明さを1つ含む反復用トイ例」を最初に設計せよ。
- 直観的な絵（エレベーター、アリ、コーヒーの粉の球など）と計算例の両方を用意し、概念ごとに少なくともどちらか一方、要所では両方を使う。

## 伏線の植え方と回収

- 序盤で生徒に良い疑問を出させ、教師が「良い問いだが、答えはお前が第◯幕で自分で見つける。保留にしておけ」と明示的に保留する。
- 回収時は生徒に「第◯幕の保留の答えが見えたお」と**明示的に**言わせる。読者が回収に気づかない伏線は無価値。
- 教材全体で2〜4本。多すぎると散漫になる。
- 「後で数学の姿を取って戻ってくる。覚えておけ」型の軽い予告も併用してよい。

## 歴史的失敗の活用

素朴な案が「実際に歴史上の一流の学者によって試され、棄却された」事実があるなら必ず使う（完成例：ラプラスの遅延重力、ノルドシュトレムのスカラー重力、アインシュタイン自身のエントヴルフ理論）。効果は2つ：(1) 素朴案が藁人形でない証明になる、(2) 生徒（読者）の失敗を「本人も2年間ここで立ち往生した」と慰め、学習の心理的安全性を作る。史実が確認できない場合は捏造せず、論理上の失敗としてのみ扱う。必要ならweb検索で裏を取る。

**後工程を安くする配慮**：ここで書く固有名詞・年号・実験値・帰属は、後で yaruo-proofread が1件ずつ出典を確認して脚注を付ける対象になる。だから検証可能な事実は**教科書級の定番（canonical なページ1枚で裏が取れるもの）**を選び、出典が見つけにくい曖昧な統計値は避ける。定番の事実ほど校正時のフェッチが1回で済み、トークンも節約できる。作成中にweb検索で裏取りした事実は、その URL を作業メモに控えておくと校正の再検索を省ける。

## キャラクター規範

**やらない夫（教師）**
- 答えを先に言わない。「計算してみろ」「数を数えろ」「反例をやる」「では判定してもらおう」で駆動する。
- 生徒の正しい着眼は短く強く認める（「鋭い」「良い言語化だ」「完璧な直観だ」）。
- 命名・整理・歴史的補足・対称性の列挙など「まとめ」だけを担当する。
- 生徒が本当に詰まったら（添字計算など）、答えでなく**次の一手だけ**を与える（「反対称ペアの向きを揃えてから縮約しろ」）。

**やる夫（生徒）**
- 語尾は「〜だお」「〜かお」「〜お」。一人称は「やる夫」。
- 素朴案を自信満々に出し（「シンプルイズベストだお」）、失敗して悔しがる（「ぐぬぬ」「うぐぐ」）。
- 過去の教訓を自力で流用する（「これさっきと同じ病気だお！」）。学習の転移を見せることが再発見感の核。
- 計算の手を動かすのは基本的に生徒。途中で詰まる様子（「添字の海で溺れてるお」）も時々入れる。
- 気づきの瞬間は「！！」「……あっ」「あーーー！」で立ち上げ、直後に内容を自分の言葉で言語化する。
- **知識予算**：生徒が使ってよい知識は、(a) 冒頭の人物設定で明示した履修範囲、(b) それまでの幕で自分が到達した内容、の2つだけ。専門的な数値・固有名詞・事件史は教師が出すか、生徒がその場の推論で導く。出所を誤魔化す台詞（「調べた記憶があるお」）で生徒に正解を言わせるのは禁止。生徒が教師より物知りになった瞬間、対話は講義の二人朗読に堕ちる。

## 文体・書式規約

- 常体。地の文はほぼ置かず、対話で運ぶ。
- 話者名は `**やる夫**：` / `**やらない夫**：` を行頭に。台詞は「」で囲み、継続行は全角スペースでインデントする。
- 数式はディスプレイ数式 `$$...$$` を基本とし、台詞中の短い式は `$...$`。
- 幕・節見出しは `## 第N幕　タイトル` / `### N-M　小見出し ── 概念名`。区切りに `---`。
- 重要な到達点は **太字**、最終結果は `\boxed{}`。
- 感情表現（！！、……、？？）は転換点に絞る。乱発すると安っぽくなる。
- 対話のテンポ：1つの台詞は長くても10行程度。教師の長広舌は分割し、間に生徒の相槌・反応を挟む。

## アンチパターン集（既刊22冊の実地レビューからの教訓）

既刊で実際に発生した失敗の型と、その処方箋。執筆中に一つでも心当たりが出たら、その場で直す。

**1. 物知りやる夫（講義の二人朗読）**
教師が問い、生徒が教科書的な正答を即答する1問1答が続くと、対話は講義を2人で分担朗読しているだけになる。生徒の正答率が高すぎる教材は失敗作。主要な問いの2〜3回に1回は、生徒に誤答・誤判定をさせてから反例で覆す。誤答は2段階で書く：まず「もっともらしい間違い」→教師が事実・数字で潰す→修正。クイズの型を生徒のメタ発言（「その聞き方をするときは、だいたい逆だお」）で種明かしするのも禁止──読者に型を意識させた時点で魔法が解ける。

**2. 教師の①②③一括開示**
概念群を教師が箇条書きで一気に開示すると、その全項目で再発見サイクルが省略される（最悪の実例：4装置を1台詞1,400字で講義し、生徒の応答が「おおお…！」の感嘆のみ）。リストのうち少なくとも1〜2項目は、データパズルや設計課題として生徒に発明させる。歴史経緯・部品表など開示自体が正当な場面でも、開示の前に予想させる（順位当て・結果当て：「どの費目が一番高いと思う？」→外させてから開示）。

**3. 同型ビートの反復**
「N個目の道具だお→潰される→反省」のような同じ拍は**最大3回まで**。4回目までに必ず型を変える：
- **役割逆転**──教師が悪い案を出して生徒に既習の道具で審査させる。または生徒を当事者役（評価される側の校長など）に降ろし、制度の抜け穴を自分で「発明」させてから、史実で答え合わせする。生徒が自分の設計した制度の加害者側を体験する構図は特に強い。
- **ミニ演習化**──反復トイ例に数値を入れて計算させ、破綻を生徒自身の手から出す。
- **誤判定→データで覆す**──「判定しろ」の連続では、時々わざと外させる。

**4. 歴史・分派・応用ツアー幕の講義化**
歴史幕は「教師のナレーション＋生徒の要約相槌」に退行しやすい。処方箋は2つ：(a) **意思決定シミュレーション**──当時の情報だけを与えて生徒に選択させ、史実で答え合わせする（「1998年のCTOとして、どの規格陣営に乗る？」）。(b) **照合予想**──各学派・各方式について「本編で歩いた筋と合う所／衝突する所」を生徒に予想させてから教師が答え合わせする。ツアー幕は全体の2割以下に抑える。

**5. 驚く係のやる夫（数値の手渡し）**
桁で驚ける数字を教師が全部提示し、生徒が驚くだけの係になっている教材は再発見感が出ない。**幕ごとに最低1回**、生徒が数行の計算をして結論の数字を自分の手で出す場面を置く。反復トイ例は宣言するだけでなく、実際に数値計算で使い倒す（宣言したのに定性利用しかしない既刊があった）。

**6. 時事依存**
現在進行形・未確定の出来事を反例の骨格に使わない。数年で古びるか、事実関係ごと覆るリスクを教材が背負う。骨格は歴史的定番（検証済みで動かない事例）に置き、時事はあくまで彩りとして「執筆時点」を明記するか脚注に落とす。

**7. 承認と感情記号のインフレ／欠乏**
教師の承認（「いい問いだ」「鋭い」）が毎回同じ言い回しで機械的に入ると、褒めの価値が下がる。承認は要所に絞る。逆に感情記号（！！、……あっ）が全編ゼロだと転換点が平板に流れる──各幕最大の気づき1箇所には必ず立ち上がりを置く。同型問答の連続（「〜できるか」「できないお」×3）は1回＋まとめで足りる。

## 品質チェックリスト（納品前に自己検査）

- [ ] 導入した概念すべてで再発見サイクルが回っているか（素朴案なしで導入された概念がないか）
- [ ] 素朴案はどれも「本当に初学者が思いつく案」か（藁人形になっていないか）
- [ ] 反例・検算に具体的な数値または数行の手計算があるか
- [ ] 新しい判定装置には合格例と不合格例の両方を与えたか
- [ ] 反復用トイ例が幕をまたいで再登場しているか
- [ ] 植えた伏線がすべて明示的に回収されているか
- [ ] 失敗の原因の言語化を教師でなく生徒がやっているか
- [ ] クライマックスで条件列挙→候補棄却→一意化の流れがあるか
- [ ] 最終幕に既知の結果・実験・極限との答え合わせがあるか
- [ ] 分量配分：道具構築の幕が全体の40〜50%か
- [ ] 総分量が5〜10万字に収まっているか（超えるなら分冊・圧縮を提案したか）
- [ ] 歴史・分派・応用のツアー幕が全体の2割以下か
- [ ] 生徒が知識予算（人物設定の履修範囲＋作中で到達した内容）を超えた知識を披露していないか
- [ ] 主要な問いに対する生徒の誤答・誤判定が定期的にあるか（全問正解の物知りになっていないか）
- [ ] 教師の①②③一括開示のうち、生徒に発明させられる項目を残していないか（開示前の予想もないか）
- [ ] 同じビートが4回以上連続していないか（役割逆転・ミニ演習・誤判定のいずれかで崩したか）
- [ ] 各幕に生徒の手計算が1回以上あるか
- [ ] 現在進行形の時事を反例の骨格にしていないか
