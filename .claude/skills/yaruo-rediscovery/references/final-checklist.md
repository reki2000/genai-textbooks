# 制作前・納品前チェック

作業冒頭に全文を読み、該当項目をプロット、概念表、執筆計画へ先に組み込む。本文完成後は同じ項目で全体を再確認し、問題を見つけたら該当箇所を直す。

## 学習設計

- [ ] 対象読者、前提知識、到達目標、扱わない範囲が一貫している。
- [ ] 核心概念では、課題、素朴な案、反例、原因の言語化、修正、検算が成立している。
- [ ] `###` の各節と独立した幕間・終幕・対話演習に、やる夫とやらない夫のやりとりが最低5往復あり、5往復未満を空疎な相槌で水増ししていない。
- [ ] 5～9往復の節でも再発見サイクルを十分な密度で描き、課題提起、正答、教師のまとめだけの節になっていない。
- [ ] 前提知識や補助用語に不要な完全サイクルを強制していない。
- [ ] 素朴な案が初学者にも一見もっともらしく、藁人形になっていない。
- [ ] 生徒が知識予算を超えた知識を突然披露していない。
- [ ] 新しい判定装置を、可能なら合格例と不合格例の両方で試している。
- [ ] 反復トイ例が実際の計算・判定に再利用されている。

## 因果とドラマ

- [ ] 各幕・節が具体的な課題や前の解決策の限界から始まる。
- [ ] 生徒が失敗の原因を言語化し、教師が答えを先に開示していない。
- [ ] 教師の質問へ生徒が正答を即答せず、迷い、想起、計算、自己修正、導出のいずれかを経ている。
- [ ] やらない夫は、やる夫の発言内容や感情をほぼ必ず受け止めてから次の質問・指示へ進み、反応なしの短文質問だけを返していない。反応表現も場面に応じて変化している。
- [ ] 正解が前の失敗を回収し、次の未解決課題へつながっている。
- [ ] 同じビート、短問短答、教師の一括開示が連続していない。
- [ ] 各節で、やる夫が簡単なアイデアから大成功を夢見て一度は調子に乗り、その題材に即した形で現実へ引き戻されている。
- [ ] 各節を教師の総括で閉じず、やる夫が発見、失敗の理由、残った疑問を自分の言葉でまとめている。
- [ ] ドラマ、コメディ、安堵が学習上の仕事を持ち、安全に描かれている。
- [ ] 伏線が保留時と回収時の両方で明示されている。

## 構成と検証

- [ ] 長編・サーベイでは、各主要幕の終盤に同形式のコラム的コーナーがあり、その幕までの道具を新しい対象へ累積適用している。
- [ ] 長編・サーベイでは、2〜4幕ごとの区切りに幕間があり、本全体でまとめ、発展、対話式理解度クイズをすべて実施している。
- [ ] コラムと幕間が要点の再掲や分量稼ぎではなく、前幕の回収、学習の転移、次幕の始動のいずれかを担っている。
- [ ] 道具構築が最も厚く、ツアー部分が全体の20%以内である。
- [ ] クライマックスで条件列挙、候補棄却、答えの絞り込みを省略していない。
- [ ] 最終幕で既知の結果、実験、極限、具体値のいずれかと照合している。
- [ ] 末尾の演習も対話形式で、各演習にやる夫の誤答または不完全な案、手掛かり、推論・計算、自己修正、正答がある。
- [ ] 歴史的事実、固有名詞、年号、実験値を検証でき、未確定の時事へ依存していない。
- [ ] 分量のための空疎・定型的な相槌、同義反復、不要な概念を削っている。必要な反応表現は後続内容と結び付き、会話の感情や理解の進展を示している。
- [ ] 話者表示、句読点、数式、見出しが `style-guide.md` の書式に従っている。
- [ ] 金額を `100万円`、`1,000円` のような半角算用数字で、割合を `40%` のような半角算用数字と半角 `%` で表記している。
- [ ] `python3 scripts/check_dialogue_constraints.py <教材.md>` が違反なしで終了する。

## 数式と計算

- [ ] 核心となる導出で、論理的に必要な途中式と各変形の理由を示している。
- [ ] 記号、添字、符号規約、単位、近似条件を必要な時点で定義している。
- [ ] 概念的な跳躍を「計算すると」で隠していない。長い機械計算を省略した場合も、出発式、操作、結果を示している。
- [ ] 中心となる展開・代入・計算を生徒が担当し、教師の長広舌だけで導出していない。
- [ ] 重要な式を、次元・単位、符号、既知の極限、特殊ケース、具体値のいずれかで検算している。
