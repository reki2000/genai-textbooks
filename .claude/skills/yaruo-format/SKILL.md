# やる夫式教材 書式整形スキル

対象は yaruo-rediscovery が生成した会話形式の教材（docs/books/*/README.md）。これは**公開前の機械lint**であり、書式の予防そのものは yaruo-rediscovery の「Markdown書式」指定が担う。本スキルの主眼は、執筆時の指定を守っても混入しやすい CommonMark/GFM のレンダリング崩れ──**処理2（強調記号 `**` の破損）と処理3（表の区切り行欠落）**──の機械的な検出・修正にある。処理1（会話表記）と処理4（発言末の句点）は、執筆時にほぼ予防される前提の保険として残す。

処理は4つで、すべてスクリプトで機械的に行う。**手作業でやらない。** 順番は 1→2→3→4（会話表記を先に確定させる）。モデルが本文を読み直して書き直すのではなく、各スクリプトがバイト単位で修正するため、崩れの検出・修正はこの機械lintが最も安いパスになる。

どのスクリプトも冪等で、`--check` を付けると変更せずに対象行を報告する（対象があれば終了コード1）。必ず --check → 適用 → `git diff --stat` で変更行数がスクリプトの報告と一致することを確認する、の順で進める。

## 処理1: 会話表記整形

話者行に続く発言はカギカッコで開かず、継続行を含む行頭の全角空白（U+3000）も削除する。単独の行末空白は削除し、Markdown の hard break を表す半角空白2個は保持する。

`**名前**：` に加えて、`**名前**（老いて）：` のように太字名とコロンの間へ注記を挟む行も話者行として扱う。各話者ブロックを上から走査し、行頭が `「` の発言区間をすべて判定する。対応する `」` を入れ子込みで探し、その `」` が行末なら発言外枠として一組を削除する。区間の前後に通常文・数式・表があっても削除対象とする。対応する `」` が行末以外にある場合は、先頭の `「` も削除しない。対応する `」` が次の話者行・節境界までにない場合は、壊れた発言外枠として先頭の `「` だけを削除し、warning を出す。

```text
「今日は晴れだ」             → 今日は晴れだ
「「今の姿」を考える」       → 「今の姿」を考える
「コールマン報告」という語   → 「コールマン報告」という語

「体積は長さの3乗だお。
一辺の長さは……」

$$L \propto V^{1/3}$$

「1899年に発売。飲める形に改造した」

    ↓

体積は長さの3乗だお。
一辺の長さは……

$$L \propto V^{1/3}$$

1899年に発売。飲める形に改造した
```

```bash
python3 .claude/skills/yaruo-format/scripts/format_dialogue.py docs/books/<file>/README.md --check
python3 .claude/skills/yaruo-format/scripts/format_dialogue.py docs/books/<file>/README.md
```

## 処理2: 強調記号 `**` の修正

CommonMark の flanking 規則により、`**` の**外側が文字（非空白・非約物）で内側が約物**（。、「」（）『』：など）に接していると、強調として解釈されず `**` がそのまま画面に表示される。日本語はスペースを入れずに書くためこの破損が頻発する。

```
壊れる例: 心理学では**快楽の踏み車（ヘドニック・トレッドミル）**と呼ぶ
          → 閉じ ** の直前が「）」（約物）、直後が「と」（文字）なので強調にならない
修正後:   心理学では**快楽の踏み車（ヘドニック・トレッドミル）** と呼ぶ
```

スクリプトは壊れている側の外側に半角スペースを1つ挿入する。正しくレンダリングされる箇所には触れない。コードブロック・インラインコード・数式内は対象外。

```bash
python3 .claude/skills/yaruo-format/scripts/fix_emphasis.py docs/books/<file>/README.md --check
python3 .claude/skills/yaruo-format/scripts/fix_emphasis.py docs/books/<file>/README.md
```

### 警告への対応

スクリプトが自動修正せず warning として報告するパターンは、該当行を目視して手動で判断する：

- **`** が奇数個`**：多くは強調が行をまたいでいるケース（開き `**` と閉じ `**` が別の行）。同一段落内なら CommonMark 上は有効なので、開き側の直後・閉じ側の直前がそれぞれ約物で壊れていないかだけ確認し、壊れていれば手で半角スペースを入れる。
- **`長さ2以外の * の連続`**：`*italic*` や `***` や区切り線。意図されたものならそのまま。
- **`** の直後（直前）が空白`**：`** 重要**` のような書き間違い。空白を内側から外側へ移す。

## 処理3: 表の区切り行の修正

GFM の表はヘッダ行の直後に区切り行（`|---|---|` 形式）が必須で、これが無いと表としてレンダリングされず `|` がそのまま画面に表示される。スクリプトは `|` で始まる行が2行以上連続するブロックを表とみなし、2行目が区切り行でなければヘッダ行の列数に合わせた区切り行を挿入する。正常な表・コードブロックや数式内・`|` 行が1行だけの箇所には触れない。

```bash
python3 .claude/skills/yaruo-format/scripts/fix_tables.py docs/books/<file>/README.md --check
python3 .claude/skills/yaruo-format/scripts/fix_tables.py docs/books/<file>/README.md
```

### 警告への対応

- **`区切り行の列数がヘッダ行と一致しない`**：GFM では列数不一致だと表にならない。該当の表を目視し、区切り行の列数をヘッダ行に合わせて手で直す。
- **`表の直前が空行でない`**：段落の続きと解釈され表にならないことがある。発言中の表でも、表の前に空行を1行入れる。

## 処理4: 発言末の句点補完

話者ブロックの最後が自然言語行で、末尾に終止約物がなければ `。` を補う。`？`、`?`、`！`、`!`、`……`、`…`、`──` の後には補わない。末尾の強調記号 `**`、脚注参照、閉じカッコの内側まで見て判定し、追加する句点は行末（Markdown記号の外側）に置く。

コードブロック、TeX数式、Markdown表、引用、箇条書きで終わる話者ブロックは対象外。読点・コロン・セミコロンで終わる場合は自動修正せず warning を出す。

```bash
python3 .claude/skills/yaruo-format/scripts/fix_dialogue_periods.py docs/books/<file>/README.md --check
python3 .claude/skills/yaruo-format/scripts/fix_dialogue_periods.py docs/books/<file>/README.md
```

## 完了報告

ファイルごとに以下を報告する：

- 会話表記整形：修正行数／warning の一覧と対応
- 強調記号修正：修正行数／警告の一覧と各warningへの対応（手動修正した・問題なしと判断した等）
- 表の修正：挿入した区切り行の数／警告の一覧と対応
- 発言末の句点補完：修正行数／warning の一覧と対応
- `git diff --stat` の行数がスクリプト報告と整合していること
